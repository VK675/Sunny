# Настройка базы данных Solário (Supabase)

Пошагово, ~5–7 минут. После этого пришли мне **2 значения** из шага 6 — и я подключу авторизацию и каталог к базе.

## 1. Создать проект
1. Зайди на **https://supabase.com** → Sign in (через GitHub/email) — бесплатно.
2. **New project**:
   - Name: `solario`
   - Database Password: придумай и **запиши** (понадобится редко, но не потеряй).
   - Region: **Europe (West)** / Frankfurt — ближе к Португалии.
3. Нажми Create. Подожди ~2 минуты, пока проект поднимется.

## 2. Создать таблицы
1. Слева **SQL Editor** → **New query**.
2. Открой файл `database/schema.sql`, скопируй **всё** содержимое, вставь в редактор.
3. Нажми **Run** (или Ctrl+Enter). Должно быть «Success».
   > Создаёт таблицы `profiles`, `regioes` (id, nome, producao_kwh_kwp, iva — заполняется сразу), `paineis`, `favoritos`, `calculos` с правилами RLS.

## 3. Загрузить каталог (66 реальных панелей)
1. SQL Editor → **New query**.
2. Открой `database/seed.sql`, скопируй всё, вставь, **Run**.
   > `seed.sql` начинается с `truncate` — старый каталог (493 сгенерированные панели) очистится и заменится на 66 реальных моделей.
3. Проверь: слева **Table Editor** → таблица `paineis` → должно быть **66** строк.

## 4. Включить вход по email
1. Слева **Authentication** → **Providers** → убедись, что **Email** включён.
2. (для удобства разработки) В настройках Email-провайдера **выключи “Confirm email”** —
   тогда регистрация сразу выполняет вход, без подтверждения почты.
   Позже, для «боевого» режима, можно включить обратно.

## 5. (для меня) Скопировать ключи
1. Слева **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** — вида `https://xxxxxxxx.supabase.co`
   - **anon public** ключ — длинная строка (Project API keys → `anon` `public`).

## 6. Прислать мне
Пришли в чат:
```
URL:  https://xxxxxxxx.supabase.co
anon: eyJhbGciOi...
```

## 7. Аккумуляторы (опционально, +1 минута)
1. SQL Editor → **New query**.
2. Открой `database/baterias.sql`, скопируй всё, вставь, **Run**.
3. Проверь: Table Editor → таблица `baterias` → **53** строки.

> Пока таблицы нет, приложение автоматически использует встроенный каталог
> (`js/baterias.js`) — всё работает и без этого шага.

## 8. Pedidos de orçamento / instalação (опционально)
1. SQL Editor → **New query**.
2. Открой `database/pedidos.sql`, скопируй всё, вставь, **Run**.
3. Проверь: Table Editor → таблица `pedidos`.

> Без этой таблицы форма заявки покажет дружелюбное сообщение «не вдалося enviar»;
> с таблицей — заявки сохраняются по пользователю (RLS), видны в «Os meus pedidos».

### ⚠️ Безопасность
- **anon-ключ публичный** — его безопасно держать во фронтенде. Доступ к данным защищён правилами RLS (Row Level Security), которые уже заданы в `schema.sql`: каждый видит только свои избранное/расчёты, каталог — только на чтение.
- **НЕ присылай** ключ `service_role` (секретный, полный доступ). Он не нужен для фронтенда.
